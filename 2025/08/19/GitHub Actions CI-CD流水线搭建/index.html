GitHub Actions CI-CD流水线搭建

Jaryujun
2025-08-19
CI/CD, Docker, GitHub Actions, Spring Boot

本文将详细介绍如何为Spring Boot项目搭建一套完整的CI/CD流水线，包括Docker容器化、GitHub Actions自动化。

流程

提交代码
  -> GitHub Actions 执行测试与构建
  -> 构建并推送 Docker 镜像
  -> 通过 SSH 登录服务器
  -> docker compose 拉取新镜像并重启服务

1. Dockerfile

示例文件：Dockerfile

FROM maven:3.9.9-eclipse-temurin-17 AS build
WORKDIR /app

COPY pom.xml .
RUN mvn -B dependency:go-offline

COPY src ./src
RUN mvn -B clean package -DskipTests

FROM eclipse-temurin:17-jre
WORKDIR /app

COPY --from=build /app/target/*.jar app.jar

EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

如果项目不是单模块结构，把 COPY 路径替换成你自己的目录即可。

2. GitHub Actions 工作流

示例文件：.github/workflows/ci-cd.yml

name: CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  test-and-build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: temurin

      - name: Cache Maven
        uses: actions/cache@v4
        with:
          path: ~/.m2/repository
          key: ${{ runner.os }}-maven-${{ hashFiles('**/pom.xml') }}
          restore-keys: |
            ${{ runner.os }}-maven-

      - name: Test
        run: mvn -B test

      - name: Package
        run: mvn -B clean package -DskipTests

  docker:
    if: github.ref == 'refs/heads/main'
    needs: test-and-build
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest

  deploy:
    if: github.ref == 'refs/heads/main'
    needs: docker
    runs-on: ubuntu-latest
    steps:
      - name: Deploy
        uses: appleboy/ssh-action@v1.2.0
        with:
          host: ${{ secrets.DEPLOY_HOST }}
          username: ${{ secrets.DEPLOY_USER }}
          key: ${{ secrets.DEPLOY_SSH_KEY }}
          script: |
            cd /opt/app
            docker compose pull
            docker compose up -d

3. 服务器端配置

示例文件：docker-compose.yml

services:
  app:
    image: ghcr.io/<your-org>/<your-repo>:latest
    container_name: app
    restart: unless-stopped
    ports:
      - "8080:8080"
    env_file:
      - .env

示例文件：.env

SPRING_PROFILES_ACTIVE=prod
SPRING_DATASOURCE_URL=<your-database-url>
SPRING_DATASOURCE_USERNAME=<your-database-username>
SPRING_DATASOURCE_PASSWORD=<your-database-password>
JWT_SECRET=<your-jwt-secret>

.env 不应提交到仓库。

4. GitHub Secrets

至少需要这些 Secrets：

  • DEPLOY_HOST
  • DEPLOY_USER
  • DEPLOY_SSH_KEY

如果镜像仓库或应用还需要其他凭证，也统一放到 Secrets 或服务器 .env 中，不要写进文章或仓库源码。

5. 使用方式

首次在服务器准备好 docker-compose.yml 和 .env 后，后续只需要向 main 分支推送代码：

git add .
git commit -m "chore: update app"
git push origin main

流水线会自动执行测试、构建镜像并部署。

安全说明

  • 本文所有主机名、路径、仓库名、变量值均为占位符。
  • 不要在工作流、docker-compose.yml、文章示例里写死真实密码或密钥。
  • 优先使用 SSH 私钥，不使用明文密码登录服务器。
