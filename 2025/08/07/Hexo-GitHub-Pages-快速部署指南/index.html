Hexo + GitHub Pages 快速部署指南

一、安装 Hexo 博客系统

1. 安装依赖环境

  • Node.js ≥ 16.x
  • Git ≥ 2.x

2. 安装 Hexo CLI

npm install -g hexo-cli

3. 初始化博客项目

hexo init my-blog
cd my-blog
npm install

二、安装 Minima 主题

1. 克隆主题到 themes 目录

cd themes
git clone https://github.com/hexo-blog/hexo-theme-minima.git minima

2. 设置主题

修改根目录下 _config.yml：

theme: minima

3. 安装渲染器（如需要）

npm install hexo-renderer-pug hexo-renderer-stylus --save

三、创建与预览文章

hexo new "你好，Hexo"
hexo clean
hexo g
hexo s

访问本地预览：http://localhost:4000

四、部署到 GitHub Pages（推荐双仓库方式）

🛠 准备工作

  1. 创建两个仓库：
仓库 用途
hexo-blog 存放 Hexo 源码
hexo-blog.github.io 存放生成的静态页面（部署用）
  1. 安装部署插件：
npm install hexo-deployer-git --save
  1. 配置 _config.yml：
deploy:
  type: git
  repo: https://github.com/hexo-blog/hexo-blog.github.io.git
  branch: main

部署命令

hexo clean && hexo generate && hexo deploy

执行成功后访问：

🔗 https://hexo-blog.github.io/

五、常用命令速查表

命令 说明
hexo init 初始化博客
hexo new "标题" 新建文章
hexo g 生成静态文件
hexo s 本地预览
hexo d 发布部署

六、自动化脚本（可选）

在 package.json 添加：

"scripts": {
  "cbd": "hexo clean && hexo generate && hexo deploy"
}

执行一条命令即可发布：

npm run cbd

七、文件结构示意

D:\projects
├── hexo-blog-blog         # Hexo 源码目录（博客编辑）
│   ├── _config.yml
│   ├── themes/minima/
│   └── ...
└── hexo-blog.github.io    # 静态部署目录（自动生成）
